# Release: v0.1.2 Trace and Metric Semantics Fixes

Date: 2026-03-24
Type: patch

## Summary
This patch resolves trace/log correlation and metric semantic consistency issues identified in review.

## Changes
1. HTTP middleware now extracts W3C trace context from request headers before starting spans and logs `trace_id` from the actual started span context.
2. HTTP middleware now sets `X-Trace-Id` response header from the active span trace ID.
3. Worker dead-letter metric behavior is unified as snapshot semantics only:
   1. `asynq_dead_letter_total` is updated via `ObserveQueueSnapshot`.
   2. Middleware failure path no longer mutates this gauge.
4. Legacy readiness check behavior is hardened:
   1. When a legacy `Fn` check is already running, readiness fails fast with an explicit error.
   2. Existing timeout guard remains in place for response latency control.

## Validation
1. `cd module && go test -race ./...`
2. `make test`
3. `make lint`
