# Release: v0.1.1 Observability Contract Fixes

Date: 2026-03-24
Type: patch

## Summary
This patch aligns module behavior with the repository observability contracts after review findings.

## Changes
1. Worker trace continuity now accepts valid async parent context from payload fields:
   1. Supports W3C `traceparent`.
   2. Supports explicit `trace_id` + `span_id`.
2. Worker metrics contract expanded in `module/workerx`:
   1. Added `ObserveEnqueue` helper for `asynq_jobs_enqueued_total`.
   2. Added `ObserveQueueSnapshot` helper for `asynq_queue_depth`, `asynq_queue_oldest_age_seconds`, and `asynq_dead_letter_total`.
3. Readiness behavior hardened:
   1. `FnCtx` checks run with timeout and context cancellation.
   2. Legacy `Fn` checks now use bounded wait-time and single-flight execution to prevent indefinite request blocking and per-request goroutine accumulation.
4. Bootstrap OTLP behavior aligned with env contract:
   1. Added `http/protobuf` exporter support.
   2. Added sampler selection support using `OTEL_TRACES_SAMPLER` and `OTEL_TRACES_SAMPLER_ARG`.
5. Example integration updated to consume module worker metrics helpers instead of duplicating Asynq metric registration.

## Validation
1. `cd module && go test -race ./...`
2. `make test`
3. `make lint`
4. `make smoke`

## Compatibility
1. No breaking changes to required public module APIs.
2. Added helper APIs in `workerx` are additive.
