# Release: v0.1.4 Hardening Batch (D10)

Date: 2026-03-24
Type: minor hardening

## Summary
This release completes a broad hardening pass across readiness, worker metrics, trace continuity, alert validation, and CI policy gates.

## Delivered Items
1. Readiness migration and policy:
   1. In-repo readiness usage/tests are now `FnCtx` based.
   2. Documented legacy `Fn` deprecation deadline: **2026-06-30**.
   3. Added `scripts/check-readiness-fn-deprecation.sh` and wired it into `make lint`.
2. Readiness concurrency tests:
   1. Added parallel probe tests for slow dependencies and mixed check sets.
3. Worker metric behavior tests:
   1. Added multi-run lifecycle metric tests.
   2. Added explicit helper `workerx.ObserveRetried(...)` for contract-level retry metric emission.
4. Trace continuity:
   1. Added integration test for HTTP -> enqueue payload -> worker trace continuity.
   2. Improved traceparent parsing to honor sampled-bit semantics from any valid flags byte.
5. Cardinality stability:
   1. Added load-oriented tests for HTTP route label stability (`_unmatched` behavior).
   2. Added load-oriented worker label stability tests for `queue` and `task_type`.
6. Metric contract enforcement:
   1. Added module-level test asserting all required metric family names from AGENTS contract are emitted by module APIs.
7. Alert semantics tests:
   1. Added `alerts/tests/baseline.rules.test.yml` with realistic gauge/counter streams.
   2. Added `scripts/test-alert-rules.sh` running `promtool test rules` (local binary or docker fallback).
   3. Updated `DeadLetterGrowth` to gauge-compatible `delta(...)` semantics.
8. CI race gate:
   1. Added `scripts/test-race.sh`.
   2. Updated `module-ci.yml` and `policy-ci.yml` to run race-test gate.
