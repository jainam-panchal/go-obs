# Release: v0.1.3 Alert, Readiness, and Logging Hardening

Date: 2026-03-24
Type: patch

## Summary
This patch aligns dead-letter alert semantics with gauge behavior, improves legacy readiness behavior under concurrent probes, and makes logging serialization failures explicit.

## Changes
1. Dead-letter alert rule updated for gauge snapshot semantics:
   1. Replaced `increase(asynq_dead_letter_total[30m])` with `(max_over_time(...) - min_over_time(...)) > 0`.
2. Legacy readiness check path improved:
   1. Keeps single-flight guard for legacy `Fn` checks.
   2. While a legacy check is running, returns recent cached result (within stale window) when available to reduce readiness flapping.
3. Logging helpers hardened:
   1. `ginx` and `workerx` now log explicit `log_marshal_failed` errors when JSON marshal fails instead of silently ignoring errors.
